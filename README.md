## 💡 Trik Rahasia VS Code: Mengintip Hasil Sebelum di-Push

Kamu tidak perlu melakukan `git push` berkali-kali ke GitHub hanya untuk melihat apakah tulisan Markdown milikmu sudah rapi atau belum. 

Saat kamu sedang membuka file `README.md` di VS Code, tekan kombinasi tombol ini di keyboardmu secara bersamaan:

> **`Ctrl + Shift + V`** (Untuk Windows) atau **`Cmd + Shift + V`** (Untuk Mac)

VS Code akan langsung membuka jendela baru di sebelahmu (*Live Preview*) yang menampilkan wujud asli dokumen tersebut saat nanti dipajang di GitHub. Sangat praktis!

Jika file `README.md` ini sudah kamu simpan, jangan lupa lakukan ritual `git add .`, `git commit -m "Menambahkan file README.md"`, dan `git push` agar brosur keren websitemu ini bisa langsung dibaca oleh seluruh dunia di halaman GitHub-mu!

Apakah penjelasan tentang cara membuat brosur kode dengan Markdown ini cukup mudah kamu pahami? Jika sudah aman, kita bisa lanjut ke langkah ke-24 untuk belajar memberikan label versi pada proyek kita (*Git Tagging*)!
